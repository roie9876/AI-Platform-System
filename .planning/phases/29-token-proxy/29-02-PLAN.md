**1. Create `k8s/base/token-proxy/deployment.yaml`:**

Follow the api-gateway Deployment pattern with these differences:
- `name: token-proxy` (label `app: token-proxy`)
- `replicas: 2` (HA by default, per D-12)
- `image: ${ACR_SERVER}/aiplatform-token-proxy:latest`
- `containerPort: 8080` (not 8000)
- All health probes target port `8080`
- `SERVICE_NAME` env value: `"llm-proxy"`
- Environment variables needed (minimal set — proxy only needs Cosmos DB access):
  - `SERVICE_NAME` = `"llm-proxy"`
  - `AZURE_OPENAI_BASE` with value `"https://ai-platform-system.openai.azure.com"` (can be overridden via configmap later)
  - `COSMOS_ENDPOINT` from secretKeyRef `aiplatform-secrets` key `COSMOS_ENDPOINT`
  - `AZURE_CLIENT_ID` from secretKeyRef `aiplatform-secrets` key `AZURE_CLIENT_ID`
  - `AZURE_TENANT_ID` from secretKeyRef `aiplatform-secrets` key `AZURE_TENANT_ID`
  - `AZURE_WORKLOAD_CLIENT_ID` from secretKeyRef `aiplatform-secrets` key `AZURE_WORKLOAD_CLIENT_ID`
  - `APPLICATIONINSIGHTS_CONNECTION_STRING` from secretKeyRef `aiplatform-secrets` key `APPLICATIONINSIGHTS_CONNECTION_STRING`
- `envFrom: configMapRef: aiplatform-config`
- Resources: `requests: {cpu: 100m, memory: 128Mi}`, `limits: {cpu: 500m, memory: 256Mi}` (per research §7.3 — lighter than api-gateway since proxy is simple passthrough)
- `serviceAccountName: aiplatform-workload`
- `azure.workload.identity/use: "true"` label on pod template
- Volume mount for secrets-store CSI (same as api-gateway)
- Liveness probe: `/healthz` port 8080, `initialDelaySeconds: 5`, `periodSeconds: 10`
- Readiness probe: `/readyz` port 8080, `initialDelaySeconds: 10`, `periodSeconds: 5`
- Startup probe: `/startupz` port 8080, `initialDelaySeconds: 3`, `failureThreshold: 30`, `periodSeconds: 2`

  name: token-proxy

    apiVersion: apps/v1
    kind: Deployment
    name: token-proxy
  minReplicas: 2
  maxReplicas: 5

        name: cpu
        target:
