  <action>
Create two shell scripts in the project root `scripts/` directory:

**scripts/validate-deployment.sh** — Pre-deployment checks (run before deploy):
1. Check prerequisites: `az`, `kubectl`, `docker`, `kustomize` are installed
2. Verify Azure login: `az account show` succeeds
3. Validate Bicep: `az bicep build --file infra/main.bicep --stdout > /dev/null`
4. Validate K8s manifests: `kubectl kustomize k8s/base > /dev/null`
5. Docker build test (dry-run): For each of the 5 microservices, run `docker build --check` or `docker build --no-cache -f backend/microservices/{svc}/Dockerfile backend/ -t test-{svc}:validate` (build only, no push). Use `--target` or just build to verify Dockerfiles parse.
6. Check required environment variables or GitHub secrets documentation
7. Print summary of pass/fail per check
8. Exit with non-zero if any check fails
9. Add `set -euo pipefail` at top, use colored output

**scripts/deploy.sh** — Full end-to-end deployment orchestration:
1. Accept arguments: `--resource-group <rg>` (required), `--environment <env>` (default: prod), `--skip-infra` (skip Bicep), `--skip-build` (skip Docker), `--dry-run` (validate only)
2. Parse arguments with a while-case loop
3. Step 1 — Infrastructure: `az deployment group create --resource-group $RG --template-file infra/main.bicep --parameters infra/parameters/${ENV}.bicepparam`
4. Step 2 — Capture outputs: Parse `az deployment group show` JSON output to extract `acrLoginServer`, `aksClusterName`, `cosmosEndpoint`, `keyVaultUri` using `jq`
5. Step 3 — Configure AKS context: `az aks get-credentials --resource-group $RG --name $AKS_CLUSTER --overwrite-existing`
6. Step 4 — ACR login: `az acr login --name $ACR_NAME`
7. Step 5 — Build and push images: For each of the 5 microservices (`api-gateway`, `agent-executor`, `workflow-engine`, `tool-executor`, `mcp-proxy`), build with `docker build -t $ACR_SERVER/aiplatform-{svc}:$(git rev-parse --short HEAD) -t $ACR_SERVER/aiplatform-{svc}:latest -f backend/microservices/{svc_dir}/Dockerfile backend/` and push both tags. Also build and push frontend.
8. Step 6 — Update K8s configmap: Use `sed` to replace `REPLACE_WITH_APP_INSIGHTS_CONNECTION_STRING` with the actual value from `az monitor app-insights component show`
9. Step 7 — Update K8s secrets: Use `sed` to populate `${WORKLOAD_IDENTITY_CLIENT_ID}`, `${KEY_VAULT_NAME}`, `${AZURE_TENANT_ID}` from Bicep outputs / `az` commands
10. Step 8 — Deploy to AKS: `cd k8s/base && kustomize edit set image` for all 5 services with the git SHA tag, then `kubectl apply -k .`
11. Step 9 — Wait for rollouts: `kubectl rollout status deployment/{svc} --timeout=300s` for each service
12. Step 10 — Run smoke tests: `bash k8s/scripts/smoke-test.sh default`
13. Step 11 — Provision default tenant: `bash k8s/scripts/setup-tenant.sh default`
14. Print deployment summary with URLs and status
15. `--dry-run` mode should run validate-deployment.sh and print what would be executed without doing it
16. Use colored output and step banners for readability
17. Map service directory names correctly: `api-gateway` → `api_gateway`, `agent-executor` → `agent_executor`, etc. (underscore in filesystem, hyphen in Docker image names)

Both scripts must be executable (`chmod +x`).
  </action>
