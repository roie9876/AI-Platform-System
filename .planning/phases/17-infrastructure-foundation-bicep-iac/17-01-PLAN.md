Create `infra/modules/vnet.bicep`:

Parameters:
- `location` (string)
- `environmentName` (string, default 'prod')
- `vnetAddressPrefix` (string, default '10.0.0.0/16')
- `aksNodesSubnetPrefix` (string, default '10.0.0.0/22')
- `aksPodsSubnetPrefix` (string, default '10.0.4.0/22')
- `privateEndpointsSubnetPrefix` (string, default '10.0.8.0/24')

Resources:
- VNet named `stumsft-aiplatform-${environmentName}-vnet` with address space `vnetAddressPrefix`
- Three subnets (per D-05):
  - `aks-nodes` — for AKS node pool VMs, address prefix `aksNodesSubnetPrefix`, NSG attached
  - `aks-pods` — for Azure CNI Overlay pod IPs, address prefix `aksPodsSubnetPrefix`, NSG attached
  - `private-endpoints` — for future private endpoints, address prefix `privateEndpointsSubnetPrefix`, NSG attached
- Three NSGs (one per subnet) named `stumsft-aiplatform-${environmentName}-nsg-{subnet-name}` with default deny-all inbound, allow-all outbound rules
- All resources use public access (per D-06) — no private endpoint configuration

Outputs:
- `vnetId` (string) — resource ID of the VNet
- `vnetName` (string) — name of the VNet
- `aksNodesSubnetId` (string) — resource ID of aks-nodes subnet
- `aksPodsSubnetId` (string) — resource ID of aks-pods subnet
- `privateEndpointsSubnetId` (string) — resource ID of private-endpoints subnet

Create `infra/modules/loganalytics.bicep`:

Parameters:
- `location` (string)
- `environmentName` (string, default 'prod')
- `retentionInDays` (int, default 30)

Resources:
- Log Analytics workspace named `stumsft-aiplatform-${environmentName}-law` with SKU `PerGB2018`
- Retention set to `retentionInDays`

Outputs:
- `workspaceId` (string) — resource ID
- `workspaceName` (string) — name
- `workspaceCustomerId` (string) — customer ID for diagnostic settings
  </action>
  <verify>

Create `infra/modules/identity.bicep`:

Parameters:
- `location` (string)
- `environmentName` (string, default 'prod')

Resources:
- User-assigned Managed Identity named `stumsft-aiplatform-${environmentName}-workload-id` for workload identity federation (per INFRA-07)
  - This identity will be federated with AKS OIDC issuer in the AKS module (Plan 03)
- User-assigned Managed Identity named `stumsft-aiplatform-${environmentName}-aks-id` for AKS control plane identity

Outputs:
- `aksIdentityId` (string) — resource ID of AKS control plane identity
- `aksIdentityPrincipalId` (string) — principal ID (for role assignments)
- `workloadIdentityId` (string) — resource ID of workload identity
- `workloadIdentityPrincipalId` (string) — principal ID (for role assignments)
- `workloadIdentityClientId` (string) — client ID (for pod identity binding)

Create `infra/modules/acr.bicep`:

Parameters:
- `location` (string)
- `environmentName` (string, default 'prod')
- `sku` (string, default 'Standard')
- `aksIdentityPrincipalId` (string) — principal ID for AcrPull role assignment

Resources:
- ACR named `stumsftaiplatform${environmentName}acr` (no hyphens — ACR names must be alphanumeric, per Azure constraint) with `adminUserEnabled: false` (per INFRA-04)
- Role assignment: `AcrPull` role (GUID `7f951dda-4ed3-4680-a7ca-43fe172d538d`) on ACR scope, assigned to `aksIdentityPrincipalId` (per INFRA-04)

Outputs:
- `acrId` (string) — resource ID
- `acrLoginServer` (string) — login server URL (e.g., `stumsftaiplatformprodacr.azurecr.io`)
- `acrName` (string) — name
