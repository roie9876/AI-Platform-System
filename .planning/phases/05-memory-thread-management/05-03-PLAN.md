<!-- Thread API from Plan 05-01 -->

Thread API endpoints:
```
POST   /api/v1/threads              body: { agent_id, title? }  → ThreadResponse
GET    /api/v1/threads?agent_id=X   → { threads: ThreadResponse[], total: int }
GET    /api/v1/threads/{id}         → ThreadResponse
GET    /api/v1/threads/{id}/messages → { messages: ThreadMessageResponse[], total: int }
DELETE /api/v1/threads/{id}         → 204
PATCH  /api/v1/threads/{id}         body: { title }  → ThreadResponse
```

ThreadResponse shape:
```typescript
interface Thread {
  id: string;
  title: string | null;
  agent_id: string;
  user_id: string;
  tenant_id: string;
  is_active: boolean;
  created_at: string;
  updated_at: string;
  message_count: number;
  last_message_preview: string | null;
}
```

ThreadMessageResponse shape:
```typescript
interface ThreadMessage {
  id: string;
  thread_id: string;
  role: string;
  content: string;
  message_metadata: Record<string, unknown> | null;
  sequence_number: number;
  created_at: string;
}
```

     - Call `POST /api/v1/threads` with `{ agent_id: agentId }` to create a thread.
     - Set activeThreadId to the new thread's ID.
     - Clear chatMessages.
     - Increment sidebarRefreshKey to refresh thread list.

  3. **Select Thread flow** (onSelectThread):
     - Set activeThreadId.
     - Fetch `GET /api/v1/threads/{threadId}/messages`.
     - Map ThreadMessage[] to ChatMessage[] (role + content + sources from message_metadata).
     - Set chatMessages to the loaded history.

  4. **Delete Thread flow** (onDeleteThread):
     - Call `DELETE /api/v1/threads/{threadId}`.
     - If deleted thread was active, clear activeThreadId and chatMessages.
     - Increment sidebarRefreshKey.

  5. **Update handleChatSend**:
     - If no activeThreadId, auto-create a thread first (POST /api/v1/threads), then set activeThreadId.
     - Include `thread_id: activeThreadId` in the chat request body.
     - Remove `conversation_history` from the request when thread_id is set (server loads history from DB).
     - After streaming completes, increment sidebarRefreshKey to update thread list (title may have been set).

  6. **Update ChatSidebar usage**: Pass activeThreadId, onSelectThread, onDeleteThread, agentId, refreshKey props.

  Keep backward compatibility: if thread creation fails for any reason, fall back to stateless mode (send conversation_history as before).
