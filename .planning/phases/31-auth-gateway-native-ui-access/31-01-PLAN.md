Create `backend/microservices/auth_gateway/__init__.py` (empty).

Create `backend/microservices/auth_gateway/main.py` with a FastAPI app implementing:

**1. Configuration (env vars):**
- `AGENTS_DOMAIN` — required, the domain (e.g. `myagents.com`). Agent subdomains are `agent-{slug}.agents.{domain}`.
- `ENTRA_APP_CLIENT_ID` — from CSI secrets (same as platform)
- `ENTRA_CLIENT_SECRET` — from CSI secrets (Key Vault `entra-client-secret`)
- `AZURE_TENANT_ID` — from configmap
- `COSMOS_ENDPOINT` — from CSI secrets
- `COSMOS_DATABASE` — from configmap (default `aiplatform`)
- `SESSION_TTL` — default 3600 seconds
- `AGENT_CACHE_TTL` — default 60 seconds
- `COOKIE_SECRET` — for signing session cookies (from env or generate random on startup)

**2. Lifespan context manager:**
- Initialize `msal.ConfidentialClientApplication` with `client_id=ENTRA_APP_CLIENT_ID`, `client_credential=ENTRA_CLIENT_SECRET`, `authority=f"https://login.microsoftonline.com/{AZURE_TENANT_ID}"`
- Initialize `httpx.AsyncClient` with 120s timeout, connection pooling
- Initialize Cosmos DB client (reuse `app.repositories.cosmos_client.get_cosmos_client`)
- Get reference to `agents` container from Cosmos
- Shutdown: close httpx client, close Cosmos client
- Configure structured logging (reuse `app.core.logging_config.configure_logging`)
- Init telemetry (reuse `app.core.telemetry.init_telemetry`)

**3. Session management (in-memory):**
- `_sessions: dict[str, tuple[dict, float]]` — session_id → (user_context, expires_at)
- `create_session(user_context: dict) -> str` — generates `secrets.token_urlsafe(32)`, stores with TTL, returns session_id
- `get_session(session_id: str) -> dict | None` — returns user_context if not expired, else None
- `_gc_sessions()` — remove expired entries, called on create
- Session cookie name: `_agents_session`
- Cookie signing: use `itsdangerous.URLSafeTimedSerializer(COOKIE_SECRET)` to sign/unsign session_id

**4. Agent resolution with TTL cache:**
- `_agent_cache: dict[str, tuple[dict, float]]` — agent_slug → (agent_doc, expires_at)
- `resolve_agent(agent_slug: str) -> dict | None` — checks cache first, on miss queries Cosmos `agents` container by `slug` field (or `id`), caches result with `AGENT_CACHE_TTL`
- Returns agent document with at minimum: `id`, `tenant_id`, `slug`, `openclaw_instance_name`
- On cache miss, query: `container.query_items(query="SELECT * FROM c WHERE c.slug = @slug OR c.id = @slug", parameters=[{"name": "@slug", "value": agent_slug}])`

**5. Tenant resolution (reuse pattern from tenant.py):**
- `_tenant_cache: dict[str, tuple[str, float]]` — tenant_id → (tenant_slug, expires_at)
- `resolve_tenant_slug(tenant_id: str) -> str | None` — queries Cosmos `tenants` container

**6. OIDC routes:**

`GET /auth/login` — Initiates OIDC flow:
- Extract `return_to` from query params (the original agent URL the user wanted)
- Generate auth URL via `msal_app.get_authorization_request_url(scopes=["openid", "profile", "email"], redirect_uri=f"https://agents.{AGENTS_DOMAIN}/auth/callback", state=json.dumps({"return_to": return_to}))`
- Return `RedirectResponse` to auth URL

`GET /auth/callback` — Handles OIDC callback:
- Extract `code` and `state` from query params
- Exchange code: `msal_app.acquire_token_by_authorization_code(code=code, scopes=["openid", "profile", "email"], redirect_uri=f"https://agents.{AGENTS_DOMAIN}/auth/callback")`
- If error in result: return 401 HTML error page
- Extract claims from `result["id_token_claims"]`
- Build user_context using `extract_user_context(claims)` from `app.core.security`
- Create session, sign session_id with itsdangerous
- Parse `return_to` from state JSON
- Return `RedirectResponse` to `return_to` URL with `Set-Cookie` header:
  - name: `_agents_session`, value: signed_session_id
  - domain: `.agents.{AGENTS_DOMAIN}`, path: `/`
  - httpOnly: True, secure: True, samesite: "lax"
  - max_age: SESSION_TTL

`GET /auth/logout` — Clears session:
- Delete session from `_sessions`
- Clear cookie
- Redirect to Entra ID logout URL

**7. Health endpoints:**
- `GET /healthz` — returns 200 `{"status": "ok"}`
- `GET /readyz` — returns 200 if Cosmos client connected
- `GET /startupz` — returns 200

**8. Main proxy middleware/catch-all route:**

`@app.api_route("/{path:path}", methods=["GET", "POST", "PUT", "DELETE", "PATCH", "OPTIONS", "HEAD"])`
`@app.websocket("/{path:path}")`

For HTTP requests:
- Extract agent slug from `Host` header: parse `agent-{slug}.agents.{domain}` → `{slug}`
- If host is `agents.{AGENTS_DOMAIN}` (base domain, not agent subdomain): only serve `/auth/*` routes, return 404 for anything else
- Check session cookie `_agents_session`: unsign with itsdangerous, get session
- If no valid session: redirect to `/auth/login?return_to={current_url}`
- Resolve agent from slug via `resolve_agent()`
- If agent not found: return 404 HTML error page ("Agent not found")
- Resolve tenant slug from agent's `tenant_id`
- **Tenant access check:** compare session's `user_context["tenant_id"]` against agent's `tenant_id`. If mismatch AND user doesn't have `platform_admin` role: return 403 HTML error page ("Access denied — this agent belongs to another tenant")
- Construct pod URL: `http://oc-{openclaw_instance_name}.tenant-{tenant_slug}.svc.cluster.local:18789`
- Proxy the request via httpx: forward method, path, headers (add `X-Forwarded-For`, `X-Real-IP`, `X-Forwarded-Proto`), body
- Return the proxied response with original status code and headers

For WebSocket requests:
- Same auth/resolution flow as HTTP (extract slug, check cookie, resolve agent, check tenant)
- If no valid session: close WebSocket with 1008 (policy violation)
- Connect to pod WebSocket: `ws://oc-{openclaw_instance_name}.tenant-{tenant_slug}.svc.cluster.local:18789{path}`
- Bidirectional relay: `asyncio.gather(client_to_pod(), pod_to_client())`
- Handle disconnection gracefully (cancel pending tasks)

**9. Error pages:**
- Simple HTML responses for 403 and 404 — include a link back to the platform UI
- Use inline styles, no external CSS dependencies

**10. Add to requirements.txt:** Add `msal>=1.31.0`, `itsdangerous>=2.1.0`, `websockets>=12.0` (if not already present). `httpx` and `python-jose` are already in requirements.

**IMPORTANT implementation notes:**
- Use `from app.core.security import validate_entra_token, extract_user_context` for Bearer token path (API callers)
- Use `from app.core.config import settings` to get AZURE_TENANT_ID etc.
- The OIDC browser flow uses MSAL ConfidentialClient (separate from Bearer token validation)
- Port 8000 (matching other microservices): `CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]`
- The health router pattern: `from app.health import health_router; app.include_router(health_router)`
- Per D-09: Reuse same Entra ID app registration (ENTRA_APP_CLIENT_ID)
- Per D-05: Server-side OIDC, OpenClaw UI has no knowledge of auth
- Per D-06: In-memory sessions, no Redis
- Per D-07: Wildcard httpOnly cookie on `.agents.{domain}`
