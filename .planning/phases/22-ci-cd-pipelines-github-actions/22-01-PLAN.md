  <name>Task 1: Create GitHub Actions build-push workflow</name>
  <files>.github/workflows/build-push.yml</files>
  <action>
Create GitHub Actions workflow `.github/workflows/build-push.yml`:

**Trigger**: Push to `main` branch, only when backend/ or frontend/ files change (path filter).

**Jobs**:

**build-backend** — Matrix strategy for 5 services: [api-gateway, agent-executor, workflow-engine, tool-executor, mcp-proxy]
Steps:
1. `actions/checkout@v4`
2. `azure/login@v2` with OIDC federated credentials (uses `AZURE_CLIENT_ID`, `AZURE_TENANT_ID`, `AZURE_SUBSCRIPTION_ID` secrets)
3. `azure/docker-login@v2` with ACR login — `loginServer: ${{ secrets.ACR_LOGIN_SERVER }}`
4. Build Docker image with context `backend/` and Dockerfile `backend/microservices/${{ matrix.service }}/Dockerfile`
5. Tag with both `${{ github.sha }}` and `latest`
6. Push both tags to ACR: `${{ secrets.ACR_LOGIN_SERVER }}/aiplatform-${{ matrix.service }}`
7. Output image digest for deployment reference

**build-frontend**:
Steps:
1. `actions/checkout@v4`
2. `azure/login@v2` with OIDC
3. `azure/docker-login@v2`
4. Build Docker image from `frontend/Dockerfile` with context `frontend/`
5. Tag with `${{ github.sha }}` and `latest`
6. Push to ACR: `${{ secrets.ACR_LOGIN_SERVER }}/aiplatform-frontend`

**Required secrets** (documented in workflow comments):
- `AZURE_CLIENT_ID` — Entra ID app registration for GitHub Actions
- `AZURE_TENANT_ID` — Entra tenant
- `AZURE_SUBSCRIPTION_ID` — Azure subscription
- `ACR_LOGIN_SERVER` — e.g. stumsftaiplatformprodacr.azurecr.io

Use `permissions: id-token: write, contents: read` for OIDC.

**SecretProviderClass** (k8s/base/secrets/secret-provider-class.yaml):
Create a SecretProviderClass for Azure Key Vault CSI driver:

```yaml
apiVersion: secrets-store.csi.x-k8s.io/v1
kind: SecretProviderClass
metadata:
  name: aiplatform-keyvault
spec:
  provider: azure
  parameters:
    usePodIdentity: "false"
    useVMManagedIdentity: "false"
    clientID: "${WORKLOAD_IDENTITY_CLIENT_ID}"
    keyvaultName: "${KEY_VAULT_NAME}"
    tenantId: "${AZURE_TENANT_ID}"
    objects: |
      array:
        - |
          objectName: cosmos-endpoint
          objectType: secret
        - |
          objectName: cosmos-key
          objectType: secret
        - |
          objectName: entra-client-id
          objectType: secret
        - |
          objectName: entra-tenant-id
          objectType: secret
  secretObjects:
    - secretName: aiplatform-secrets
      type: Opaque
      data:
        - objectName: cosmos-endpoint
          key: COSMOS_ENDPOINT
        - objectName: cosmos-key
          key: COSMOS_KEY
        - objectName: entra-client-id
          key: AZURE_CLIENT_ID
        - objectName: entra-tenant-id
          key: AZURE_TENANT_ID
```

**Update kustomization.yaml** — add `secrets/secret-provider-class.yaml` to resources list.

**Note:** The placeholder variables (${...}) are replaced by Kustomize overlays or the tenant setup script at deploy time.
