**Deploy workflow** (.github/workflows/deploy.yml):

**Trigger**: `workflow_run` — runs after `build-push.yml` completes successfully on main. Also supports `workflow_dispatch` with inputs for manual deploy (environment, image_tag).

**Permissions**: `id-token: write, contents: read`

**Job: deploy**
Steps:
1. `actions/checkout@v4`
2. `azure/login@v2` with OIDC
3. `azure/aks-set-context@v4` — set kubectl context to AKS cluster (uses `AKS_CLUSTER_NAME`, `AKS_RESOURCE_GROUP` secrets)
4. Set image tag — use `${{ github.event.workflow_run.head_sha }}` (from workflow_run) or `${{ github.event.inputs.image_tag }}` (from dispatch), default to `${{ github.sha }}`
5. Deploy base platform services:
   ```
   cd k8s/base
   kustomize edit set image \
     stumsftaiplatformprodacr.azurecr.io/aiplatform-api-gateway=${{ secrets.ACR_LOGIN_SERVER }}/aiplatform-api-gateway:${IMAGE_TAG} \
     stumsftaiplatformprodacr.azurecr.io/aiplatform-agent-executor=${{ secrets.ACR_LOGIN_SERVER }}/aiplatform-agent-executor:${IMAGE_TAG} \
     stumsftaiplatformprodacr.azurecr.io/aiplatform-workflow-engine=${{ secrets.ACR_LOGIN_SERVER }}/aiplatform-workflow-engine:${IMAGE_TAG} \
     stumsftaiplatformprodacr.azurecr.io/aiplatform-tool-executor=${{ secrets.ACR_LOGIN_SERVER }}/aiplatform-tool-executor:${IMAGE_TAG} \
     stumsftaiplatformprodacr.azurecr.io/aiplatform-mcp-proxy=${{ secrets.ACR_LOGIN_SERVER }}/aiplatform-mcp-proxy:${IMAGE_TAG}
   kubectl apply -k .
   ```
6. Wait for rollout: `kubectl rollout status deployment/{service} --timeout=300s` for each service
7. Run smoke tests: `bash k8s/scripts/smoke-test.sh`

**Required secrets**: `AKS_CLUSTER_NAME`, `AKS_RESOURCE_GROUP` (plus OIDC secrets from build-push)

**Smoke test script** (k8s/scripts/smoke-test.sh):
```bash
#!/bin/bash
set -euo pipefail

SERVICES=("api-gateway" "agent-executor" "workflow-engine" "tool-executor" "mcp-proxy")
NAMESPACE="${1:-default}"
FAILED=0

echo "Running post-deploy smoke tests in namespace: ${NAMESPACE}"

for SVC in "${SERVICES[@]}"; do
  echo -n "Checking ${SVC}... "
  POD=$(kubectl get pods -n "${NAMESPACE}" -l app="${SVC}" -o jsonpath='{.items[0].metadata.name}' 2>/dev/null)
  if [ -z "$POD" ]; then
    echo "FAIL — no pod found"
    FAILED=$((FAILED + 1))
    continue
  fi

  HEALTH=$(kubectl exec -n "${NAMESPACE}" "${POD}" -- curl -s -o /dev/null -w '%{http_code}' http://localhost:8000/healthz 2>/dev/null || echo "000")
  READY=$(kubectl exec -n "${NAMESPACE}" "${POD}" -- curl -s -o /dev/null -w '%{http_code}' http://localhost:8000/readyz 2>/dev/null || echo "000")

  if [ "$HEALTH" = "200" ] && [ "$READY" = "200" ]; then
    echo "OK (health=${HEALTH}, ready=${READY})"
  else
    echo "FAIL (health=${HEALTH}, ready=${READY})"
    FAILED=$((FAILED + 1))
  fi
done

if [ $FAILED -gt 0 ]; then
  echo "SMOKE TESTS FAILED: ${FAILED}/${#SERVICES[@]} services unhealthy"
  exit 1
fi

echo "ALL SMOKE TESTS PASSED"
```
Make executable: `chmod +x k8s/scripts/smoke-test.sh`

**Tenant deploy workflow** (.github/workflows/deploy-tenant.yml):

**Trigger**: `workflow_dispatch` only — manual tenant deployment with inputs:
- `tenant_slug` (required): Tenant slug for namespace selection
- `image_tag` (optional, default: latest): Image tag to deploy

**Steps**:
1. `actions/checkout@v4`
2. `azure/login@v2` with OIDC
3. `azure/aks-set-context@v4`
4. Compute namespace: `TENANT_NS=tenant-${tenant_slug}`
5. Verify namespace exists: `kubectl get namespace ${TENANT_NS}` (fail if not found)
6. Deploy tenant overlay with image override:
   ```
   WORK_DIR=$(mktemp -d)
   cp -r k8s/overlays/tenant-template/* ${WORK_DIR}/
   find ${WORK_DIR} -type f -name "*.yaml" -exec sed -i "s/TENANT_NAMESPACE/${TENANT_NS}/g" {} \;
   find ${WORK_DIR} -type f -name "*.yaml" -exec sed -i "s/TENANT_SLUG/${tenant_slug}/g" {} \;
   cd ${WORK_DIR}
   kustomize edit set image <image-overrides-same-as-deploy.yml>
   kubectl apply -k .
   rm -rf ${WORK_DIR}
   ```
7. Wait for rollout in tenant namespace
8. Run smoke tests in tenant namespace: `bash k8s/scripts/smoke-test.sh ${TENANT_NS}`
