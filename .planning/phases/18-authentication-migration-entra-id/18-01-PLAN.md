<objective>
Replace the existing cookie-based JWT authentication with Microsoft Entra ID token validation on the backend. Create RBAC dependencies and migrate all API endpoints to use Bearer token auth.

Purpose: The current auth uses HS256 self-signed JWTs in httpOnly cookies. Entra ID provides enterprise SSO, centralized identity, and Managed Identity for service-to-service auth — required before microservice extraction in Phase 20.

Output: Backend validates Entra ID tokens, extracts tenant context from claims, enforces RBAC roles, and all 20+ API route files use the new auth dependencies.
</objective>
