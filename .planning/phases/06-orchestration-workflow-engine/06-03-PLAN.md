<!-- API contracts from Plans 01 and 02 -->

Workflow CRUD API (Plan 01):
```
POST   /api/v1/workflows                              → WorkflowDetailResponse (201)
GET    /api/v1/workflows                              → WorkflowListResponse
GET    /api/v1/workflows/{id}                         → WorkflowDetailResponse (includes nodes + edges)
PUT    /api/v1/workflows/{id}                         → WorkflowResponse
DELETE /api/v1/workflows/{id}                         → 204
POST   /api/v1/workflows/{id}/nodes                   → WorkflowNodeResponse (201)
DELETE /api/v1/workflows/{id}/nodes/{node_id}         → 204
POST   /api/v1/workflows/{id}/edges                   → WorkflowEdgeResponse (201)
DELETE /api/v1/workflows/{id}/edges/{edge_id}         → 204
GET    /api/v1/workflows/{id}/executions              → WorkflowExecutionListResponse
```

Workflow Execution API (Plan 02):
```
POST   /api/v1/workflows/{id}/execute                 → WorkflowExecutionResponse (202)
GET    /api/v1/workflows/{id}/executions/{exec_id}    → WorkflowExecutionDetailResponse
POST   /api/v1/workflows/{id}/executions/{exec_id}/cancel → WorkflowExecutionResponse
```

Key response shapes:
```typescript
interface WorkflowResponse {
  id: string; name: string; description?: string;
  workflow_type: string; is_active: boolean;
  tenant_id: string; created_by: string;
  created_at: string; updated_at: string;
}

interface WorkflowNodeResponse {
  id: string; workflow_id: string; agent_id: string;
  name: string; node_type: string;
  position_x: number; position_y: number;
  config?: Record<string, unknown>;
  execution_order: number;
  created_at: string; updated_at: string;
}

interface WorkflowEdgeResponse {
  id: string; workflow_id: string;
  source_node_id: string; target_node_id: string;
  edge_type: string; condition?: Record<string, unknown>;
  output_mapping?: Record<string, unknown>;
  created_at: string; updated_at: string;
}

interface WorkflowDetailResponse extends WorkflowResponse {
  nodes: WorkflowNodeResponse[];
  edges: WorkflowEdgeResponse[];
}

interface WorkflowExecutionResponse {
  id: string; workflow_id: string; status: string;
  started_at?: string; completed_at?: string;
  input_data?: Record<string, unknown>;
  output_data?: Record<string, unknown>;
  error?: string; tenant_id: string;
  triggered_by: string; thread_id?: string;
}

interface WorkflowNodeExecutionResponse {
  id: string; node_id: string; agent_id: string;
  status: string; started_at?: string; completed_at?: string;
  input_data?: Record<string, unknown>;
  output_data?: Record<string, unknown>;
  error?: string; thread_id?: string;
}

interface WorkflowExecutionDetailResponse extends WorkflowExecutionResponse {
  node_executions: WorkflowNodeExecutionResponse[];
}
```

**Create `frontend/src/components/workflow/execution-monitor.tsx`** — Execution status component:
- Props: `executionId: string, workflowId: string`
- Poll execution status every 2 seconds via `GET /api/v1/workflows/{workflowId}/executions/{executionId}` (useEffect with setInterval)
- Stop polling when status is "completed", "failed", or "cancelled"
- Display execution header: status badge (color-coded: green=completed, yellow=running, red=failed, gray=pending/cancelled), started_at, duration
- Display node execution list as a vertical timeline:
  - Each node shows: node name (from node_id lookup), status badge, duration, input/output preview
  - Running nodes show a spinner animation
  - Failed nodes show error message in red
  - Completed nodes show output_data.response (truncated to 200 chars with expand toggle)
- Bottom section: aggregated output_data when workflow completes
- Cancel button (visible when status="running"): POST to `/api/v1/workflows/{workflowId}/executions/{executionId}/cancel`
