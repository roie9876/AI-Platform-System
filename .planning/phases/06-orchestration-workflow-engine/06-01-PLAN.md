**Create `backend/app/api/v1/workflows.py`** with a FastAPI router:

Endpoints (all tenant-isolated using `get_tenant_id` dependency):

1. `POST /` — Create workflow. Accepts WorkflowCreateRequest. Creates Workflow row, then if `nodes` provided, creates WorkflowNode rows (validating each agent_id exists in tenant). If `edges` provided, creates WorkflowEdge rows (validating source_node_id and target_node_id exist). Returns WorkflowDetailResponse with 201 status.

2. `GET /` — List workflows. Returns WorkflowListResponse, ordered by created_at desc.

3. `GET /{workflow_id}` — Get workflow detail. Returns WorkflowDetailResponse including nodes and edges. Load nodes via `select(WorkflowNode).where(WorkflowNode.workflow_id == workflow_id).order_by(WorkflowNode.execution_order)`. Load edges via `select(WorkflowEdge).where(WorkflowEdge.workflow_id == workflow_id)`.

4. `PUT /{workflow_id}` — Update workflow metadata (name, description, workflow_type). Returns WorkflowResponse.

5. `DELETE /{workflow_id}` — Delete workflow (cascades to nodes and edges). Returns 204.

6. `POST /{workflow_id}/nodes` — Add node to workflow. Accepts WorkflowNodeCreateRequest. Validates agent_id exists in tenant. Returns WorkflowNodeResponse with 201.

7. `DELETE /{workflow_id}/nodes/{node_id}` — Remove node from workflow (cascades edges). Returns 204.

8. `POST /{workflow_id}/edges` — Add edge. Accepts WorkflowEdgeCreateRequest. Validates both node IDs belong to this workflow. Returns WorkflowEdgeResponse with 201.

9. `DELETE /{workflow_id}/edges/{edge_id}` — Remove edge. Returns 204.

10. `GET /{workflow_id}/executions` — List executions for a workflow. Returns WorkflowExecutionListResponse.

Follow the exact same pattern as agents.py for auth dependencies: `current_user: User = Depends(get_current_user)`, `tenant_id: str = Depends(get_tenant_id)`.
