**Methods to implement:**

1. **`async def run(self, workflow_id: UUID, input_data: dict, user_id: UUID, tenant_id: UUID, db: AsyncSession) -> WorkflowExecution`**
   - Load workflow with nodes and edges from DB
   - Validate workflow has at least 1 node
   - Create a WorkflowExecution record (status="running", started_at=now, input_data=input_data)
   - Create a shared Thread for cross-agent context (THRD-03): `Thread(title=f"Workflow: {workflow.name}", agent_id=first_node.agent_id, user_id=user_id, tenant_id=tenant_id)`
   - Dispatch to execution strategy based on `workflow.workflow_type`:
     - `"sequential"` → `_execute_sequential()`
     - `"parallel"` → `_execute_parallel()`
     - `"autonomous"` → `_execute_autonomous()`
     - `"custom"` → `_execute_custom()` (follows edges as DAG)
   - On success: set execution status="completed", completed_at=now, output_data=final result
   - On failure: set status="failed", error=str(exception)
   - Return the WorkflowExecution

2. **`async def _execute_sequential(self, nodes: List[WorkflowNode], edges: List[WorkflowEdge], execution: WorkflowExecution, input_data: dict, user_id: UUID, tenant_id: UUID, thread_id: UUID, db: AsyncSession) -> dict`**
   - Sort nodes by `execution_order`
   - For each node in order:
     - Create WorkflowNodeExecution record (status="running", started_at=now)
     - Build input message: if first node, use original `input_data["message"]`; otherwise use previous node's output
     - Apply output_mapping from the connecting edge if present (look up edge where source_node_id=previous_node.id and target_node_id=current_node.id)
     - Load the Agent from DB
     - Call `AgentExecutionService().execute()` — collect the full response by consuming the SSE generator (parse `data:` lines, accumulate tokens, skip `[DONE]` events)
     - Update WorkflowNodeExecution: status="completed", output_data={"response": collected_text}, completed_at=now
     - If execution fails, set node status="failed", error=str(e), and raise to abort workflow
   - Return the last node's output_data

3. **`async def _execute_parallel(self, nodes: List[WorkflowNode], edges: List[WorkflowEdge], execution: WorkflowExecution, input_data: dict, user_id: UUID, tenant_id: UUID, thread_id: UUID, db: AsyncSession) -> dict`**
   - Create a WorkflowNodeExecution for each node
   - Launch all node executions concurrently using `asyncio.gather(*tasks, return_exceptions=True)`
   - Each task: load agent, call AgentExecutionService().execute(), collect response
   - Aggregate results into `{"results": [{"node_id": ..., "node_name": ..., "response": ...}, ...]}`
   - Mark failed nodes with status="failed" but don't abort the whole workflow unless all fail
   - Return aggregated results dict

4. **`async def _execute_node(self, node: WorkflowNode, node_execution: WorkflowNodeExecution, message: str, user_id: UUID, tenant_id: UUID, thread_id: UUID, db: AsyncSession) -> str`**
   - Private helper used by both sequential and parallel
   - Load Agent by node.agent_id
   - Create a new DB session for parallel safety (use `async_sessionmaker` from core.database)
   - Build conversation_history from the shared thread's messages (for cross-agent context)
   - Call `AgentExecutionService().execute(agent=agent, user_message=message, db=session, thread_id=thread_id, user_id=user_id)`
   - Consume SSE generator, parse JSON from `data:` lines, accumulate token content
   - The SSE format is `data: {"token": "...", "done": false}\n\n` — parse each line, extract "token" value, concatenate
   - Update node_execution: status="completed", output_data={"response": full_text}, completed_at=now
   - Return collected text

5. **`_apply_output_mapping(self, source_output: dict, mapping: dict) -> str`**
   - If mapping is None, return source_output["response"] as-is
   - Otherwise apply key mapping: e.g., `{"summary": "response"}` means take source_output["response"] and label it "summary" in the input to next node
   - Return the mapped message as a string

**Add to `backend/app/services/workflow_engine.py`:**

1. **`async def _execute_autonomous(self, nodes, edges, execution, input_data, user_id, tenant_id, thread_id, db) -> dict`**
   - Autonomous mode: a "router" agent decides which agents to call and in what order
   - Build a system prompt describing available agents: for each node, include name, agent description, and agent_id
   - Create a router prompt: "You are a workflow orchestrator. Given the user's request, decide which agents to call and in what order. Available agents: {agent_list}. Respond with a JSON plan: {\"steps\": [{\"agent_id\": \"...\", \"message\": \"...\"}]}. Execute the plan step by step."
   - Use the first node's agent as the router (or the node with node_type="router" if one exists)
   - Execute the router agent to get a plan
   - Parse the plan JSON from the response
   - Execute each step sequentially using `_execute_node()` with the specified agent and message
   - Collect all results and return aggregated output
   - If JSON parsing fails, fall back to sequential execution order

2. **`async def _execute_custom(self, nodes, edges, execution, input_data, user_id, tenant_id, thread_id, db) -> dict`**
   - DAG traversal following edges
   - Find root nodes (nodes with no incoming edges)
   - Use topological sort to determine execution order
   - Execute nodes level by level: nodes at the same depth run in parallel, results feed to next level via output_mapping on edges
   - For conditional edges (edge_type="conditional"), evaluate the condition against the source node's output (simple JSON key matching: condition={"key": "status", "value": "success"} checks if output["status"] == "success")
   - For error edges (edge_type="error"), only follow if source node failed
   - Return the output of terminal nodes (nodes with no outgoing edges)

3. **Sub-agent delegation support** — Add to AgentExecutionService integration:
   - When a node has `node_type="sub_agent"`, the parent agent can trigger it via a special tool
   - Add a method `async def register_sub_agent_tools(self, workflow: Workflow, nodes: List[WorkflowNode], db: AsyncSession) -> List[dict]` that creates tool schemas for each sub_agent node: `{"type": "function", "function": {"name": "delegate_to_{node_name}", "description": "Delegate subtask to {agent_name}", "parameters": {"type": "object", "properties": {"message": {"type": "string"}}}}}`
   - These tools are injected into the parent agent's tool list during workflow execution
   - When the tool is called, execute the sub-agent node via `_execute_node()`
