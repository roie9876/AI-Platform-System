    Create `docs/architecture/` directory and the unified architecture document per D-01 (single document) and D-02 (presentation-optimized diagrams).

    **Document structure (follow this ordering):**

    1. **Title & Executive Summary** — "AI Agent Platform as a Service — Architecture Design"
       - One-paragraph summary: multi-tenant PaaS for AI agent orchestration, model-agnostic, Microsoft-aligned
       - State the two architectural planes: Control Plane (management) and Runtime Plane (execution)

    2. **System Overview** — High-level description of the platform
       - Control Plane: API layer, auth, tenant management, agent CRUD, configuration
       - Runtime Plane: Agent execution engine, tool sandbox, model abstraction, memory
       - Data Layer: PostgreSQL (relational + vector via pgvector), Redis (cache + pub-sub), Azure Blob (artifacts)
       - Include **Mermaid Diagram 1: System Overview** — flowchart showing Control Plane, Runtime Plane, Data Layer as three horizontal bands with key components in each. Keep boxes to 8-12 max. Use simple labels.

    3. **Architecture Layers — Control Plane**
       - API Gateway: REST API (FastAPI), versioned under /api/v1/, OpenAPI auto-generated
       - Authentication & Authorization: JWT (httpOnly cookies), Entra ID integration, RBAC
       - Tenant Management: Multi-tenant with tenant_id in JWT, row-level isolation
       - Agent Management: CRUD, versioning, configuration, status tracking
       - Include **Mermaid Diagram 2: Control Plane Components** — block diagram showing API Gateway → Auth → Tenant Context → Feature Routers (Agents, Tools, Data Sources, Policies)

    4. **Architecture Layers — Runtime Plane**
       - Agent Execution Engine: Request → model call → tool invocation → response cycle
       - Model Abstraction Layer: OpenAI-compatible interface, multi-model routing, fallback chains, circuit breaker
       - Tool Execution Sandbox: Isolated execution, input validation, timeout handling
       - Memory System: Short-term (conversation history), long-term (vector store via pgvector)
       - Orchestration Engine: Sequential chains, parallel execution, sub-agent delegation, autonomous mode
       - Include **Mermaid Diagram 3: Agent Execution Flow** — sequence diagram showing: User → API → Agent Engine → Model Abstraction → LLM Provider → (optional) Tool Sandbox → Response. Keep to 6-8 participants max.

    5. **Architecture Layers — Data Layer**
       - PostgreSQL 16+: Relational data (tenants, users, agents, configs, policies, audit logs) + pgvector for embeddings
       - Redis 7+: Session cache, pub-sub for real-time updates, task queue backend (Celery)
       - Azure Blob Storage: File attachments, agent artifacts, large outputs
       - Azure AI Search: RAG indexing and retrieval (hybrid vector + keyword)
       - Include **Mermaid Diagram 4: Data Flow** — flowchart showing how data moves: API request → PostgreSQL (config) + Redis (cache) → Agent Engine → Model Provider + Tool Sandbox → PostgreSQL (results/audit) + Redis (streaming). Simple arrows, no more than 10 nodes.

    6. **Feature Area Subsystems** — One subsection per feature area, each with:
       - Purpose (1-2 sentences)
       - Key components and their responsibilities
       - How it interacts with other subsystems
       - Feature areas to cover (match REQUIREMENTS.md sections):
         a. Agent Management (AGNT-01 through AGNT-05)
         b. Tool Management (TOOL-01 through TOOL-04)
         c. Data Sources & RAG (DATA-01 through DATA-03)
         d. Model Abstraction & Routing (MODL-01 through MODL-04)
         e. Memory Management (MEMO-01 through MEMO-03)
         f. Thread & State Management (THRD-01 through THRD-03)
         g. Orchestration & Workflows (ORCH-01 through ORCH-05)
         h. Policy Engine & Governance (PLCY-01 through PLCY-04)
         i. Evaluation Engine (EVAL-01 through EVAL-03)
         j. Cost & Token Observability (COST-01 through COST-04)
         k. AI Services Integration (AISV-01 through AISV-03)
         l. Terminal & CLI (TERM-01, TERM-02)

    7. **Security Architecture**
       - Authentication: JWT with httpOnly cookies, Entra ID for enterprise SSO
       - Multi-tenant isolation: tenant_id in JWT, row-level filtering, namespace isolation in K8s
       - Secrets management: Azure Key Vault for model API keys, data source credentials
       - Content safety: Azure Content Safety for input/output filtering
       - Network security: API Gateway rate limiting, CORS, HTTPS-only
       - Include **Mermaid Diagram 5: Security Boundaries** — diagram showing tenant isolation layers: API Gateway → Auth Middleware → Tenant Context → Data Isolation (row-level) + Runtime Isolation (container). 6-8 nodes max.

    8. **Deployment Architecture**
       - Local development: Docker Compose (PostgreSQL, Redis, Backend, Frontend)
       - Production: AKS with namespace-per-tenant or pod isolation
       - CI/CD: GitHub Actions → ACR → AKS
       - Include **Mermaid Diagram 6: Azure Deployment Topology** — diagram showing: Users → Azure API Management → AKS (Control Plane pods + Runtime Plane pods) → Azure PostgreSQL + Azure Redis + Azure Blob + Azure AI Search + Azure OpenAI. Use cloud/service icons conceptually. Keep to 10-12 nodes max.

    9. **Microsoft Azure Service Mapping** (per D-04 and D-05)
       - Table format with columns: Platform Component | Azure Service | Recommended SKU (Dev) | Recommended SKU (Prod) | Monthly Cost Range (Dev) | Monthly Cost Range (Prod)
       - Mappings (use current Azure pricing as of 2025-2026):
         | Platform Component | Azure Service | Dev SKU | Prod SKU |
         |---|---|---|---|
         | API Compute | Azure Container Apps / AKS | AKS Free tier (dev) / B-series node | AKS Standard with D4s_v5 nodes |
         | Agent Runtime | Azure Container Apps | Consumption plan | Dedicated plan with autoscale |
         | Primary Database | Azure Database for PostgreSQL Flexible Server | Burstable B1ms | General Purpose D4s_v3 |
         | Vector Database | pgvector extension on same PostgreSQL | (included) | (included) |
         | Cache | Azure Cache for Redis | Basic C0 | Standard C1 |
         | Object Storage | Azure Blob Storage | LRS Hot | GRS Hot |
         | Identity | Microsoft Entra ID | Free tier | P1 |
         | Secrets | Azure Key Vault | Standard | Standard |
         | AI Models (default) | Azure OpenAI Service | S0 (pay-per-token) | S0 (PTU for production) |
         | Search / RAG | Azure AI Search | Free tier | Basic or S1 |
         | Monitoring | Azure Monitor + App Insights | Free tier (5GB) | Log Analytics workspace |
         | API Gateway | Azure API Management | Developer | Standard |
         | Message Queue | Azure Service Bus | Basic | Standard |
         | Container Registry | Azure Container Registry | Basic | Standard |
         | Content Safety | Azure AI Content Safety | S0 | S0 |
       - Include ballpark monthly cost ranges per D-05:
         - Dev environment: ~$150-300/month (minimal SKUs, single instance)
         - Production environment: ~$2,000-5,000/month (HA, autoscale, multiple instances)
         - Note: Azure OpenAI costs are usage-dependent and not included in base estimates

    10. **Scalability Model**
        - Horizontal scaling: AKS node autoscaling, pod HPA for API and runtime
        - Database scaling: PostgreSQL read replicas, connection pooling (PgBouncer)
        - Cache scaling: Redis clustering for high-throughput pub-sub
        - Agent runtime: Container-per-execution for isolation, queue-based scheduling

    Ensure all Mermaid diagrams use simple syntax (flowchart TD, sequenceDiagram, or graph LR). No subgraphs deeper than 2 levels. Labels should be 2-4 words max. Per D-02, optimize for projection in a large room.
