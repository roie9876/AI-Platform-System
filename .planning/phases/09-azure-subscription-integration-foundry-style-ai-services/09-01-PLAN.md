**backend/app/services/azure_arm.py:**
Create `AzureARMService` class with these methods:

```python
import httpx
from typing import List, Optional

class AzureARMService:
    ARM_BASE = "https://management.azure.com"

    async def list_subscriptions(self, access_token: str) -> list[dict]:
        """GET {ARM_BASE}/subscriptions?api-version=2022-12-01
        Returns list of {subscriptionId, displayName, tenantId, state}"""

    async def list_resources_by_type(self, access_token: str, subscription_id: str, resource_type: str) -> list[dict]:
        """GET {ARM_BASE}/subscriptions/{subscription_id}/resources?$filter=resourceType eq '{resource_type}'&api-version=2021-04-01
        Returns list of {id, name, type, location, properties}"""

    async def list_search_indexes(self, access_token: str, resource_id: str) -> list[dict]:
        """Parse resource_id to extract resourceGroupName and searchServiceName.
        GET {ARM_BASE}/{resource_id}/indexes?api-version=2024-07-01
        Returns list of index names"""
```

Use `httpx.AsyncClient` for HTTP calls. Set `Authorization: Bearer {access_token}` header. Handle pagination via `nextLink` in ARM responses. Return empty list on 403/404 (subscription may not have access). Raise `httpx.HTTPStatusError` on 5xx. Do NOT store or log the access_token.

**backend/app/api/v1/azure_subscriptions.py:**
Create router with prefix handled by router.py. Endpoints:

1. `POST /azure/subscriptions` — Connect a subscription
   - Body: `AzureSubscriptionCreate` with fields: subscription_id (str), display_name (str), tenant_azure_id (str), access_token (str), refresh_token (str, optional)
   - Encrypt access_token and refresh_token via `encrypt_api_key()` before storing
   - Create AzureSubscription row scoped to `get_tenant_id()`
   - Return `AzureSubscriptionResponse`

2. `GET /azure/subscriptions` — List connected subscriptions for tenant
   - Query AzureSubscription filtered by tenant_id
   - Return list of `AzureSubscriptionResponse`

3. `DELETE /azure/subscriptions/{subscription_db_id}` — Disconnect subscription
   - Delete AzureSubscription by id AND tenant_id (tenant scoping)
   - Cascade deletes AzureConnections via FK
   - Return 204

4. `GET /azure/subscriptions/{subscription_db_id}/resources` — Discover resources
   - Query param: `resource_type` (required, string) e.g. "Microsoft.Search/searchServices"
   - Decrypt access_token from AzureSubscription
   - Call `AzureARMService.list_resources_by_type()`
   - Return `ResourceDiscoveryResponse` with list of `DiscoveredResource`

5. `GET /azure/subscriptions/discover` — Discover subscriptions with OAuth token
   - Query param: `access_token` (required, passed via Authorization header as Bearer token — NOT query param)
   - Actually: Accept access_token in request header `X-Azure-Token` to avoid logging in URL
   - Call `AzureARMService.list_subscriptions()`
   - Return list of discovered subscriptions (not yet stored)
