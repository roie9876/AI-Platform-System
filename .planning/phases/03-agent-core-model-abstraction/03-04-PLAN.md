  <action>
    Build the chat playground modeled on Azure AI Foundry's interface (per D-03, D-04, D-05).

    **frontend/src/app/dashboard/agents/[id]/chat/page.tsx — Main chat page:**
    - "use client" component
    - Three-panel flex layout: sidebar (w-64), chat center (flex-1), config panel (w-80)
    - State: messages (array of {role, content}), isStreaming (boolean), agent config, selected model endpoint
    - Fetch agent details on mount (GET /api/v1/agents/{id})
    - Fetch available model endpoints (GET /api/v1/model-endpoints)

    SSE streaming implementation:
    - On send message, POST to /api/v1/agents/{agent_id}/chat with fetch() (NOT EventSource — need POST body)
    - Use `response.body.getReader()` + TextDecoder to read SSE chunks
    - Parse "data: {...}" lines, extract content tokens
    - Append tokens to current assistant message as they arrive (per D-09: responsive feel)
    - On done=true, finalize message
    - On error, show error in chat with retry option
    - Maintain conversation_history in state, send with each request

    **frontend/src/components/chat/chat-sidebar.tsx — Conversation sidebar (per D-03):**
    - Left panel showing conversation history list
    - For PoC: single conversation per session (no persistence yet — Phase 5 adds threads)
    - "New Chat" button that clears messages
    - Show current agent name and status at top
    - Collapsible on mobile (hamburger toggle)

    **frontend/src/components/chat/chat-messages.tsx — Chat message display:**
    - Scrollable message list with auto-scroll to bottom on new messages
    - User messages: right-aligned, blue background
    - Assistant messages: left-aligned, gray background, with agent name label
    - Streaming indicator: show a blinking cursor/dot at end of assistant message while isStreaming=true (per D-09)
    - Empty state: "Start a conversation with {agent.name}"
    - Error messages styled distinctly (red border, retry button)

    **frontend/src/components/chat/chat-input.tsx — Message input:**
    - TextArea (auto-growing, max 6 rows) with Send button
    - Send on Enter (Shift+Enter for newline)
    - Disable input + show loading state while streaming
    - "Stop" button visible during streaming (aborts the fetch via AbortController)

    **frontend/src/components/chat/config-panel.tsx — Right-side config panel (per D-04, D-05):**
    - Model endpoint selector dropdown at TOP of panel (per D-05 — like Foundry's deployment picker)
      - Populated from /api/v1/model-endpoints
      - Switching endpoint updates agent's model_endpoint_id via PUT /api/v1/agents/{id}
    - Config fields below (per D-04 — changes apply to next message):
      - System prompt (textarea, editable)
      - Temperature (slider, 0-2, step 0.1, shows current value)
      - Max tokens (number input)
      - Timeout (number input)
    - "Apply" button saves config changes via PUT /api/v1/agents/{id} — creates new config version automatically
    - Show current config version number
    - Collapsible on mobile

    Styling: Clean, professional look matching AI Foundry aesthetic — neutral grays, clear typography, subtle borders, no heavy decoration. Tailwind utility classes throughout.
