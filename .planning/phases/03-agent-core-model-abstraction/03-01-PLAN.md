  <action>
    Create three new SQLAlchemy models following existing patterns (Base, UUIDMixin, TimestampMixin, tenant_id FK):

    **backend/app/models/agent.py — Agent model:**
    - Columns: name (String 255, not null), description (Text, nullable), system_prompt (Text, nullable), status (String 50, default "inactive" — values: inactive/active/error), temperature (Float, default 0.7), max_tokens (Integer, default 1024), timeout_seconds (Integer, default 30), tenant_id (UUID FK to tenants.id, not null, indexed), model_endpoint_id (UUID FK to model_endpoints.id, nullable — agent can exist before endpoint assigned), current_config_version (Integer, default 1)
    - Per D-02: these fields match the agent creation form fields

    **backend/app/models/model_endpoint.py — ModelEndpoint model:**
    - Columns: name (String 255, not null), provider_type (String 50, not null — "azure_openai", "openai", "anthropic", "custom"), endpoint_url (String 1024, nullable — Azure OpenAI uses standard SDK URL), model_name (String 255, not null — e.g. "gpt-4o", "claude-3-opus"), api_key_encrypted (Text, nullable — encrypted API key for non-Azure providers, per D-06/D-07 local PoC stores Fernet-encrypted key in DB), auth_type (String 50, not null — "entra_id" or "api_key", per D-06), is_active (Boolean, default True), tenant_id (UUID FK to tenants.id, not null, indexed), priority (Integer, default 0 — for fallback ordering per MODL-04)
    - NOTE: For PoC, API keys are Fernet-encrypted in DB. Production would use Azure Key Vault per D-07. Add ENCRYPTION_KEY to Settings (config.py).

    **backend/app/models/agent_config_version.py — AgentConfigVersion model:**
    - Columns: agent_id (UUID FK to agents.id, not null, indexed), version_number (Integer, not null), config_snapshot (JSON, not null — stores full config at that point: system_prompt, temperature, max_tokens, timeout_seconds, model_endpoint_id), change_description (String 500, nullable), tenant_id (UUID FK to tenants.id, not null, indexed)
    - Unique constraint on (agent_id, version_number)
    - Per D-08: simple version snapshots — each config change creates a new version

    **backend/app/models/__init__.py:**
    - Add imports for Agent, AgentConfigVersion, ModelEndpoint to __all__

    **backend/alembic/versions/002_agent_model_endpoint_schema.py:**
    - Create model_endpoints table first (agents references it)
    - Create agents table with FK to model_endpoints and tenants
    - Create agent_config_versions table with FK to agents and tenants
    - Add unique constraint on (agent_id, version_number) for config versions
    - Follow exact pattern from 001_initial_schema.py (revision="002", down_revision="001")

    **backend/app/core/config.py:**
    - Add ENCRYPTION_KEY: str = "change-me-in-production-use-fernet-key" to Settings

    **backend/app/api/v1/schemas.py — Add Pydantic schemas:**

    Agent schemas:
    - AgentCreateRequest: name (str, min 1, max 255), description (str | None), system_prompt (str | None), model_endpoint_id (UUID | None), temperature (float, ge=0, le=2, default=0.7), max_tokens (int, ge=1, le=128000, default=1024), timeout_seconds (int, ge=1, le=300, default=30)
    - AgentUpdateRequest: same fields all Optional
    - AgentResponse: id, name, description, system_prompt, status, temperature, max_tokens, timeout_seconds, model_endpoint_id, current_config_version, tenant_id, created_at, updated_at — with model_config = {"from_attributes": True}
    - AgentListResponse: agents (list[AgentResponse]), total (int)

    ModelEndpoint schemas:
    - ModelEndpointCreateRequest: name (str), provider_type (str — validated to allowed values), endpoint_url (str | None), model_name (str), api_key (str | None — raw key, encrypted before storage), auth_type (str — "entra_id" or "api_key"), priority (int, default=0)
    - ModelEndpointUpdateRequest: same fields all Optional
    - ModelEndpointResponse: id, name, provider_type, endpoint_url, model_name, auth_type, is_active, priority, tenant_id, created_at, updated_at — NO api_key field exposed. model_config = {"from_attributes": True}
    - ModelEndpointListResponse: endpoints (list[ModelEndpointResponse]), total (int)

    AgentConfigVersion schemas:
    - AgentConfigVersionResponse: id, agent_id, version_number, config_snapshot (dict), change_description, created_at — model_config = {"from_attributes": True}

    **backend/app/api/v1/agents.py — Agent CRUD router:**
    - Use `router = APIRouter()`, import get_current_user from auth.py and get_tenant_id from middleware
    - All endpoints require authentication (Depends(get_current_user)) and tenant context (Depends(get_tenant_id))
    - POST / — Create agent, auto-create AgentConfigVersion v1 with initial config snapshot
    - GET / — List agents filtered by tenant_id, return AgentListResponse
    - GET /{agent_id} — Get single agent by ID, verify tenant_id matches
    - PUT /{agent_id} — Update agent, create new AgentConfigVersion with incremented version_number and updated config snapshot. Increment current_config_version.
    - DELETE /{agent_id} — Soft consideration: for PoC, hard delete agent + cascade config versions
    - GET /{agent_id}/versions — List all config versions for an agent
    - POST /{agent_id}/rollback/{version_number} — Rollback: load config_snapshot from specified version, apply to agent fields, create new version (per D-08)

    **backend/app/api/v1/model_endpoints.py — ModelEndpoint CRUD router:**
    - POST / — Create endpoint. If auth_type="api_key" and api_key provided, encrypt with Fernet before storing in api_key_encrypted. Use cryptography.fernet.Fernet with ENCRYPTION_KEY from settings.
    - GET / — List endpoints filtered by tenant_id
    - GET /{endpoint_id} — Get single endpoint, verify tenant_id
    - PUT /{endpoint_id} — Update endpoint (re-encrypt api_key if changed)
    - DELETE /{endpoint_id} — Delete endpoint

    **backend/app/api/v1/router.py — Register new routers:**
    - Add: `api_router.include_router(agents_router, prefix="/agents", tags=["agents"])`
    - Add: `api_router.include_router(model_endpoints_router, prefix="/model-endpoints", tags=["model-endpoints"])`

    SECURITY: Never return api_key or api_key_encrypted in any response. ModelEndpointResponse deliberately omits these fields.
