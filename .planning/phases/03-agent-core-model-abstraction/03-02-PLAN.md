    **backend/app/services/model_abstraction.py — ModelAbstractionService class:**

    Circuit breaker (simple in-process implementation, no external dependency):
    - Track per-endpoint failure counts in a dict (endpoint_id → {failures: int, last_failure: datetime, state: "closed"|"open"|"half_open"})
    - CLOSED: requests flow normally. After 3 consecutive failures → OPEN
    - OPEN: reject requests immediately (raise). After 60s → HALF_OPEN
    - HALF_OPEN: allow one request. If succeeds → CLOSED, reset counter. If fails → OPEN
    - Method: `is_endpoint_available(endpoint_id: UUID) -> bool`
    - Method: `record_success(endpoint_id: UUID)` / `record_failure(endpoint_id: UUID)`

    Model abstraction:
    - `async def complete(messages: list[dict], endpoint: ModelEndpoint, stream: bool = True) -> AsyncGenerator[str, None]`
    - Build LiteLLM call params:
      - For auth_type="azure_openai": use model format "azure/<model_name>", set api_base from endpoint_url. For Entra ID, use `azure-identity` DefaultAzureCredential to get token (per D-06). Set api_key to the bearer token.
      - For auth_type="api_key": decrypt api_key_encrypted via secret_store, pass to litellm as api_key. Use model format "<provider>/<model_name>" (e.g. "openai/gpt-4o", "anthropic/claude-3-opus").
    - Call `await litellm.acompletion(model=model_str, messages=messages, stream=stream, temperature=..., max_tokens=..., timeout=...)`
    - If stream=True, yield delta content chunks from the async generator
    - Wrap errors: catch litellm exceptions, record failure in circuit breaker, re-raise as a custom ModelError

    Fallback logic:
    - `async def complete_with_fallback(messages: list[dict], endpoints: list[ModelEndpoint], stream: bool = True) -> AsyncGenerator[str, None]`
    - Sort endpoints by priority (lower = higher priority)
    - Try each endpoint in order: skip if circuit breaker is open, attempt completion, on success record_success and yield
    - If endpoint fails, record_failure, log warning, try next
    - If all endpoints exhausted, raise ModelError("All model endpoints failed")
  </action>

    **backend/app/services/agent_execution.py — AgentExecutionService:**

    Core execution method:
    - `async def execute(agent: Agent, user_message: str, db: AsyncSession, conversation_history: list[dict] | None = None) -> AsyncGenerator[str, None]`
    - Steps:
      1. Load agent's assigned model_endpoint (and fallback endpoints in same tenant with same model_name, sorted by priority)
      2. Build messages list: [{"role": "system", "content": agent.system_prompt}] + conversation_history + [{"role": "user", "content": user_message}]
      3. Call model_abstraction.complete_with_fallback(messages, endpoints, stream=True)
      4. Yield SSE-formatted chunks: `data: {"content": "token", "done": false}\n\n`
      5. After stream completes, yield `data: {"content": "", "done": true}\n\n`
      6. On error, yield `data: {"error": "message", "done": true}\n\n`
    - Update agent status to "active" on successful execution start, "error" on failure (per AGNT-04)
