<task type="auto">
  <name>Task 1: Application Insights + diagnostic settings Bicep modules</name>
  <files>infra/modules/appinsights.bicep, infra/modules/alerts.bicep</files>
  <action>
1. Create infra/modules/appinsights.bicep:
   - Parameters: location, environmentName, workspaceId (from loganalytics module output)
   - Resource: Microsoft.Insights/components@2020-02-02 (type: web, kind: web)
     - name: stumsft-aiplatform-{environmentName}-appi
     - Application_Type: web
     - WorkspaceResourceId: workspaceId parameter
     - IngestionMode: LogAnalytics
     - RetentionInDays: 90
   - Outputs: appInsightsId, instrumentationKey, connectionString

2. Create infra/modules/alerts.bicep:
   - Parameters: location, environmentName, appInsightsId, aksClusterId, cosmosAccountId, actionGroupEmail (string)
   - Resource: Microsoft.Insights/actionGroups@2023-01-01
     - name: stumsft-aiplatform-{environmentName}-ag
     - emailReceivers with actionGroupEmail parameter
   - Resource: Microsoft.Insights/metricAlerts@2018-03-01 for each:
     a. "aks-pod-restart-alert" — AKS pod restart count > 5 in 5min window
        - scopes: [aksClusterId]
        - criteria: MetricName "restartingContainerCount", operator GreaterThan, threshold 5
     b. "app-5xx-rate-alert" — Application Insights 5xx rate > 10 in 5min
        - scopes: [appInsightsId]
        - criteria: MetricName "requests/failed", operator GreaterThan, threshold 10
     c. "cosmos-ru-consumption-alert" — Cosmos DB Normalized RU > 80%
        - scopes: [cosmosAccountId]
        - criteria: MetricName "NormalizedRUConsumption", operator GreaterThan, threshold 80
     d. "aks-node-cpu-alert" — AKS node CPU > 80%
        - scopes: [aksClusterId]
        - criteria: MetricName "node_cpu_usage_percentage", operator GreaterThan, threshold 80
   - All alerts: severity 2, evaluationFrequency PT5M, windowSize PT5M, enabled true, autoMitigate true
  </action>
  <verify>
    <automated>cd infra && az bicep build --file modules/appinsights.bicep 2>&1 | head -5 && az bicep build --file modules/alerts.bicep 2>&1 | head -5 && echo "Bicep build OK"</automated>
  </verify>
  <done>Application Insights Bicep module outputs connectionString; alerts module creates 4 metric alert rules + action group</done>
</task>
