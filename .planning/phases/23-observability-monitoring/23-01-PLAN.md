Actually let's simplify: make TelemetryMiddleware set attributes in the response path (after call_next), or better yet, use a different approach — have TelemetryMiddleware grab tenant_id from request.state after call_next returns. This way it can be outermost:

```python
class TelemetryMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        response = await call_next(request)
        # After inner middleware (TenantMiddleware) ran, tenant_id is set
        span = trace.get_current_span()
        tenant_id = getattr(request.state, "tenant_id", None)
        if tenant_id and span.is_recording():
            span.set_attribute("tenant_id", tenant_id)
        return response
```

This way order doesn't matter — TelemetryMiddleware always reads after TenantMiddleware has processed. Add it after TenantMiddleware line in each file.
