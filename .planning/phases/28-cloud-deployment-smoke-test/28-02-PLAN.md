  <action>
Create `scripts/post-deploy-config.sh` that extracts Bicep deployment outputs and populates K8s manifest placeholders.

**Arguments:** `--resource-group <rg>` (required), `--deployment-name <name>` (default: main)

**Script flow:**

1. `set -euo pipefail`, argument parsing with while-case loop
2. Extract Bicep deployment outputs via:
   ```bash
   OUTPUTS=$(az deployment group show --resource-group "$RG" --name "$DEPLOY_NAME" --query properties.outputs -o json)
   ACR_SERVER=$(echo "$OUTPUTS" | jq -r '.acrLoginServer.value')
   AKS_CLUSTER=$(echo "$OUTPUTS" | jq -r '.aksClusterName.value')
   COSMOS_ENDPOINT=$(echo "$OUTPUTS" | jq -r '.cosmosEndpoint.value')
   KEY_VAULT_URI=$(echo "$OUTPUTS" | jq -r '.keyVaultUri.value')
   ```
3. Get App Insights connection string:
   ```bash
   APP_INSIGHTS_CS=$(az monitor app-insights component show --resource-group "$RG" --query "[0].connectionString" -o tsv 2>/dev/null || echo "NOT_CONFIGURED")
   ```
4. Get workload identity client ID from managed identity:
   ```bash
   WORKLOAD_ID=$(az identity show --resource-group "$RG" --name "stu-msft-aiplatform-${ENV}-workload-id" --query clientId -o tsv 2>/dev/null || echo "PLACEHOLDER")
   ```
5. Get Azure tenant ID:
   ```bash
   TENANT_ID=$(az account show --query tenantId -o tsv)
   ```
6. Extract Key Vault name from URI:
   ```bash
   KV_NAME=$(echo "$KEY_VAULT_URI" | sed 's|https://||;s|\.vault\.azure\.net.*||')
   ```
7. Update `k8s/base/configmap.yaml`:
   - Replace `REPLACE_WITH_APP_INSIGHTS_CONNECTION_STRING` with actual value
8. Update `k8s/base/secrets/secret-provider-class.yaml`:
   - Replace `${WORKLOAD_IDENTITY_CLIENT_ID}` with actual workload identity client ID
   - Replace `${KEY_VAULT_NAME}` with actual Key Vault name
   - Replace `${AZURE_TENANT_ID}` with actual tenant ID
9. Print summary of all extracted values and updated files
10. Make idempotent — if values already replaced (no placeholder found), skip with info message

**Critical:** Use `sed -i ''` on macOS (empty string for backup), document this or use a portable approach with a temp file.

  <action>
Enhance the existing `k8s/scripts/smoke-test.sh` to add API-level validation beyond health checks.

**Keep the existing health check logic** (healthz + readyz for all 5 services) unchanged at the top.

**Add new sections after health checks:**

1. **API Endpoint Reachability** (via kubectl port-forward to api-gateway):
   - `curl -s -o /dev/null -w '%{http_code}' http://localhost:8000/api/v1/agents` — expect 200 or 401 (auth required is fine, means route exists)
   - `curl -s -o /dev/null -w '%{http_code}' http://localhost:8000/docs` — expect 200 (OpenAPI docs accessible)
   - Port-forward in background, kill on cleanup
   - Accept 2xx or 401 as "reachable" (401 means auth is enforcing, route exists)

2. **Inter-service DNS resolution** (validates K8s service DNS):
   - For each service, exec into api-gateway pod and `nslookup {service-name}` — should resolve
   - Services to check: `agent-executor`, `workflow-engine`, `tool-executor`, `mcp-proxy`

3. **Pod resource compliance**:
   - `kubectl top pods -n $NAMESPACE` — verify pods are running within resource limits
   - Parse output and warn if any pod uses >80% of its limit

4. **Summary output** with three sections: Health Checks, API Endpoints, DNS Resolution — each showing pass/fail counts

**Accept `--extended` flag** for the new checks. Without `--extended`, only run the original health checks (backward compatible). With `--extended`, run all checks.

**Accept `--ingress-url <url>` flag** for testing through actual ingress instead of port-forward.

Keep `set -euo pipefail`, exit codes: 0 = all pass, 1 = health failures, 2 = extended check failures (non-blocking).
