<objective>
Build the observability dashboard frontend inspired by the Azure AI Gateway Dev Portal: KPI tiles with trend indicators, token usage charts, cost breakdown views, execution log viewer, and shared analytics toolbar. Uses Recharts for charting.

Purpose: Provides the visual analytics experience matching the AI Gateway Dev Portal look and feel, powered by our backend observability APIs.
Output: Dashboard page, token analytics page, cost analytics page, execution logs page, shared components.

**Design Reference:** Azure-Samples/ai-gateway-dev-portal patterns:
- 4 KPI tiles (requests, tokens, latency, availability) with big numbers + trend arrows
- Recharts line/area charts with interactive legends
- Shared analytics toolbar (time range presets, auto-refresh, filters)
- 2-column chart grid layout
- Dark-compatible styling with our purple (#7C3AED) brand color
</objective>

    2. Create `frontend/src/components/observability/kpi-tiles.tsx`:
    Reusable KPI tile component inspired by AI Gateway Dev Portal dashboard tiles.

    ```tsx
    interface KpiTileProps {
      title: string;
      value: string | number;
      unit: string;
      icon: React.ReactNode;
      trend?: { direction: "up" | "down" | "flat"; pct: number };
      trendGood?: "up" | "down";  // which direction is good (up for requests, down for latency)
      subtitle?: string;
      onClick?: () => void;
      colorClass?: string;  // "text-blue-500", "text-purple-500", etc.
    }
    ```

    Render as a card with:
    - Icon in top-left with colorClass bg circle
    - subtitle in top-right corner (smaller text)
    - Big value center (text-3xl font-bold)
    - unit below value (text-sm text-gray-500)
    - Trend arrow at bottom: green up arrow or red down arrow with pct%
    - Hover state with shadow, cursor-pointer if onClick
    - Border-l-4 with colorClass

    Export `KpiTileGrid` component that wraps 4 tiles in a responsive grid:
    `grid grid-cols-1 md:grid-cols-2 xl:grid-cols-4 gap-4`

    3. Create `frontend/src/components/observability/analytics-toolbar.tsx`:
    Shared toolbar for all analytics pages.

    ```tsx
    interface AnalyticsToolbarProps {
      timeRange: string;
      onTimeRangeChange: (range: string) => void;
      granularity?: string;
      onGranularityChange?: (gran: string) => void;
      onRefresh: () => void;
      loading?: boolean;
      extra?: React.ReactNode;  // additional filter controls
    }
    ```

    Features:
    - Time range presets: buttons for "1h", "24h", "7d", "30d"
    - Granularity selector (optional): "auto", "5m", "1h", "1d"
    - Refresh button with spinning animation when loading
    - Last updated timestamp
    - Slot for extra controls (agent filter dropdown, model filter)
    - Styled with white bg, rounded border, px-4 py-2, flex items-center gap-3

    4. Create `frontend/src/components/observability/chart-card.tsx`:
    Wrapper for chart sections.

    ```tsx
    interface ChartCardProps {
      title: string;
      loading?: boolean;
      children: React.ReactNode;
      className?: string;
    }
    ```

    Render: white bg card with title h3 at top, content area with min-height, loading overlay with spinner.
