    1. Create `frontend/src/app/dashboard/evaluations/page.tsx` — Test Suite List:

    **Layout:**
    - Header: "Evaluations" title + "Create Test Suite" button (opens modal/inline form)
    - Filter bar: Agent selector dropdown (loads agents from /api/v1/agents)
    - Table/card list of test suites:
      - Name, Agent name, # test cases, last run date, last run score (colored: green >0.7, yellow 0.4-0.7, red <0.4)
      - Click to navigate to /dashboard/evaluations/{suiteId}
    - If no test suites: empty state with "Create your first test suite" prompt

    **Create Test Suite modal:**
    - Fields: Name (required), Description (optional), Agent (dropdown, required)
    - POST /api/v1/evaluations/test-suites

    2. Create `frontend/src/app/dashboard/evaluations/[suiteId]/page.tsx` — Suite Detail:

    **Layout:**
    - Header: Suite name + agent name + "Run Evaluation" button (POST /test-suites/{id}/run)
    - Two sections:

    **Test Cases section:**
    - Table: Input Message | Expected Output | Keywords | Actions(edit/delete)
    - "Add Test Case" button → inline form: input_message (required), expected_output (optional), expected_keywords (comma-separated)
    - POST /api/v1/evaluations/test-suites/{id}/cases

    **Run History section:**
    - Table of past runs: Date, Status (badge), Total Cases, Passed, Failed, Avg Score, Avg Latency
    - Click run → navigate to /dashboard/evaluations/runs/{runId}
    - Checkbox selection for 2 runs → "Compare Selected" button

    **When "Run Evaluation" clicked:**
    - Show running state (spinner + "Evaluating...")
    - Poll GET /evaluations/runs/{run_id} until status is "completed" or "failed"
    - Refresh run history on completion

    3. Create `frontend/src/app/dashboard/evaluations/runs/[runId]/page.tsx` — Run Results:

    **Layout:**
    - Header: "Run #{date}" + status badge + summary stats
    - Summary bar: Total Cases, Passed (green), Failed (red), Avg Score, Avg Latency, Total Tokens
    - Results table:
      - Input Message | Expected Output | Actual Output | Score (progress bar 0-1) | Latency | Status (pass/fail badge)
      - Click row to expand: full actual_output text, detailed metrics (similarity_score, keyword_match_rate, input_tokens, output_tokens)
    - If comparison mode (2 run_ids in query params):
      - Side-by-side per-case results
      - Delta indicators: ↑ improved, ↓ regressed
      - Summary comparison bar chart (Recharts)
