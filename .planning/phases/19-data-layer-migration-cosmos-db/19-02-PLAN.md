For each of the 5 API route files, apply the same migration pattern:

**Migration pattern (apply to each file):**

1. **Remove imports:**
   - Delete `from sqlalchemy import select, func` (and any other sqlalchemy imports)
   - Delete `from sqlalchemy.ext.asyncio import AsyncSession`
   - Delete `from app.core.database import get_db`
   - Delete `from app.models.{model} import {Model}` for all model imports

2. **Add imports:**
   - `from app.repositories.{appropriate_repo} import {Repository}`

3. **Instantiate repositories at module level:**
   - e.g., `agent_repo = AgentRepository()` after imports

4. **Rewrite endpoint signatures:**
   - Remove `db: AsyncSession = Depends(get_db)` parameter
   - Keep `current_user: dict = Depends(get_current_user)` and `tenant_id: str = Depends(get_tenant_id)`

5. **Rewrite endpoint bodies — CRUD mapping:**

   | SQLAlchemy | Cosmos DB Repository |
   |------------|---------------------|
   | `db.add(model); await db.flush()` | `result = await repo.create(tenant_id, item_dict)` |
   | `result = await db.execute(select(Model).where(...))` | `result = await repo.get(tenant_id, item_id)` or `await repo.query(tenant_id, ...)` |
   | `result.scalar_one_or_none()` | Direct return from `repo.get()` (returns None if not found) |
   | `result.scalars().all()` | `await repo.list_all(tenant_id)` or `await repo.list_by_tenant(tenant_id)` |
   | `setattr(obj, field, value); await db.flush()` | Read item, update dict fields, `await repo.update(tenant_id, id, updated_dict)` |
   | `await db.delete(obj)` | `await repo.delete(tenant_id, item_id)` |
   | `await db.refresh(obj)` | Not needed — Cosmos returns updated item from create/update |

6. **Response construction:**
   - SQLAlchemy returns ORM objects with attributes (e.g., `agent.name`)
   - Cosmos DB returns dicts (e.g., `agent["name"]`)
   - Response models (Pydantic schemas) accept dicts via `model_validate()` or `**dict`
   - For response_model to work, construct response dicts that match the Pydantic schema field names
   - Key field mappings: SQLAlchemy `id` (UUID) → Cosmos `id` (string). Ensure response returns `id` as string.

**Specific file notes:**

- **agents.py:** Replace Agent model operations. Agent create must also create AgentConfigVersion in the `agent_config_versions` container. Agent update must increment version and create new config version. Delete must also delete config versions (query + delete each).

- **tools.py:** Replace Tool and AgentTool model operations. `attach_tool` creates an AgentTool doc, `detach_tool` deletes it. Check uniqueness via `agent_tool_repo.get_by_agent_and_tool()` before creating.

- **threads.py:** Replace Thread and ThreadMessage operations. List threads for agent, create thread, list messages for thread. Messages ordered by `created_at`.

- **chat.py:** This file uses AgentExecutionService which still uses SQLAlchemy — for now, only migrate the direct DB operations in chat.py (thread creation, message listing). The service migration happens in Plan 03.

- **model_endpoints.py:** Replace ModelEndpoint CRUD. Simple entity, straightforward migration.
