**1. Create `backend/scripts/migrate_to_cosmos.py`:**

Migration script that reads all data from PostgreSQL and writes to Cosmos DB:

```python
"""
Data Migration: PostgreSQL → Cosmos DB

Reads all data from existing PostgreSQL database via SQLAlchemy
and writes to Cosmos DB containers via the repository layer.

Usage:
    cd backend
    python -m scripts.migrate_to_cosmos --database-url postgresql+asyncpg://...
"""
import asyncio
import argparse
import logging
from datetime import datetime
from uuid import UUID

from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker, AsyncSession
from sqlalchemy import select, text

# Import all SQLAlchemy models for reading
from app.models import *
from app.repositories.cosmos_client import get_cosmos_container, close_cosmos_client

logger = logging.getLogger(__name__)
```

Key functions:
- `serialize_row(row)` — Convert SQLAlchemy model instance to dict: UUID → str, datetime → ISO 8601 string, handle JSONB fields
- `async def migrate_table(session, model_class, container_name)` — Read all rows, serialize, write to Cosmos container via `container.upsert_item()`
- `async def main(database_url)` — Connect to PostgreSQL, iterate all model classes, call `migrate_table` for each
- Table-to-container mapping dict matching the 35 containers from cosmos.bicep
- Progress logging: `Migrating {table}... {count} items`
- Error handling: log failures per item, continue migration
- Summary at end: total items migrated, failures

The script keeps SQLAlchemy imports because it reads FROM PostgreSQL. It's a one-time migration tool.
