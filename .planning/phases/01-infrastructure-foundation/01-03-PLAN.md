  <action>
    Create `infra/modules/monitoring.bicep` — Observability infrastructure per D-23 through D-27:

    **Parameters:**
    - `environment`, `location`, `naming` (object), `tags` (object)
    - `retentionInDays` (int, default: 30 for dev, 90 for prod — passed via parameter)

    **Log Analytics Workspace** (`Microsoft.OperationalInsights/workspaces`):
    - SKU: PerGB2018
    - Retention: `retentionInDays` parameter
    - Daily cap: 5 GB for dev, uncapped for prod (use `dailyQuotaGb` property, -1 for unlimited)
    - Features: enable Container Insights solution

    **Application Insights** (`Microsoft.Insights/components`):
    - Kind: web
    - Application type: web
    - Linked to Log Analytics workspace (workspace-based, NOT classic)
    - `WorkspaceResourceId`: reference to Log Analytics workspace
    - Sampling: enabled (default adaptive sampling)
    - Retention: same as Log Analytics

    **Action Group** (`Microsoft.Insights/actionGroups`):
    - Name: `{naming}-platform-alerts`
    - Short name: `PlatAlerts`
    - Email receivers: parameterized (empty default — configured per environment)

    **Alert Rules** (`Microsoft.Insights/metricAlerts`) — per HLD §6 alerting rules:
    1. **AKS Node CPU > 80%** (WARNING):
       - Target: AKS cluster resource
       - Metric: node_cpu_usage_percentage
       - Threshold: 80, operator: GreaterThan, aggregation: Average, period: PT5M
       - But since AKS resource ID may not be available in this module, create these as OPTIONAL with a conditional parameter `aksClusterId` (string, default: '')

    2. **Key Vault Availability < 99%** (CRITICAL):
       - Conditional on `keyVaultId` parameter
       - Metric: Availability
       - Threshold: 99, operator: LessThan

    3. **Container Restart Count > 5** (WARNING):
       - Conditional on `aksClusterId`
       - Metric namespace: Insights.Container/pods
       - Metric: restartingContainerCount

    Make all alert rules conditional: only created when the target resource ID parameter is non-empty. This allows the module to be deployed before or after compute resources.

    **Diagnostic Settings Module** — Create a reusable child module or output template pattern:
    - Output the `logAnalyticsWorkspaceId` so other modules can send diagnostics
    - Document the pattern: each resource module should add its own diagnostic settings using this workspace ID

    **Outputs:**
    - `logAnalyticsWorkspaceId` (string) — consumed by AKS OMS agent, diagnostic settings
    - `appInsightsInstrumentationKey` (string) — for application telemetry
    - `appInsightsConnectionString` (string) — preferred connection method
    - `actionGroupId` (string) — for alert routing
  </action>

    **1. infra/main.bicep** — Top-level orchestrator (subscription-scoped deployment):

    `targetScope = 'subscription'` (deploy at subscription level to create resource groups)

    **Parameters:**
    - `environment` (string, allowed: ['dev', 'staging', 'prod'])
    - `location` (string, default: 'eastus2' per D-01)
    - `projectName` (string, default: 'aip')
    - `tags` (object, default: {})
    - `tenantId` (string — Azure AD tenant ID)
    - `alertEmailReceivers` (array of objects, default: [])
    - `monitoringRetentionDays` (int, default: 30)

    **Resource Group:**
    - `Microsoft.Resources/resourceGroups` — name: `rg-{projectName}-{environment}`
    - Location: `location`
    - Tags: merged with environment tag

    **Module composition** (all scoped to the resource group using `scope: resourceGroup(rg.name)`):

    Order matters — wire outputs as inputs:

    ```
    1. shared → outputs: naming, defaultTags
    2. monitoring → inputs: naming, tags
                 → outputs: logAnalyticsWorkspaceId, appInsightsConnectionString
    3. networking → inputs: naming, tags
                  → outputs: subnet IDs, DNS zone IDs
    4. firewall → inputs: naming, tags, firewallSubnetId (from networking)
                → outputs: firewallPrivateIp, routeTableId
    5. acr → inputs: naming, tags, privateEndpointSubnetId (from networking), acrDnsZoneId (from networking)
           → outputs: acrId, acrLoginServer
    6. aks → inputs: naming, tags, aksSystemSubnetId (from networking), aksUserSubnetId (from networking),
             logAnalyticsWorkspaceId (from monitoring), acrId (from acr)
           → outputs: kubeletIdentityObjectId, oidcIssuerUrl
    7. keyvault → inputs: naming, tags, privateEndpointSubnetId (from networking), kvDnsZoneId (from networking),
                  tenantId, aksKubeletIdentityObjectId (from aks)
               → outputs: keyVaultUri, keyVaultName
    ```

    Each module reference:
    ```bicep
    module shared './modules/shared.bicep' = {
      name: 'shared-${environment}'
      scope: resourceGroup(rg.name)
      params: { environment: environment, location: location, projectName: projectName, tags: tags }
    }
    ```
    (similar pattern for each module)

    **Top-level outputs:**
    - `resourceGroupName`
    - `aksClusterName`
    - `acrLoginServer`
    - `keyVaultUri`
    - `appInsightsConnectionString`
    - `oidcIssuerUrl`

    **2. infra/parameters/dev.bicepparam:**
    ```bicep
    using '../main.bicep'

    param environment = 'dev'
    param location = 'eastus2'
    param projectName = 'aip'
    param tenantId = '' // Set via CI/CD or local override
    param monitoringRetentionDays = 30
    param tags = {
      Environment: 'dev'
      Project: 'AI Platform'
      CostCenter: 'engineering'
    }
    ```

    **3. infra/parameters/staging.bicepparam:**
    - Same structure, `environment = 'staging'`
    - `monitoringRetentionDays = 60`

    **4. infra/parameters/prod.bicepparam:**
    - Same structure, `environment = 'prod'`
    - `monitoringRetentionDays = 90`
    - Additional tags: `Compliance: 'required'`, `BackupPolicy: 'daily'`

    NOTE: `tenantId` left empty in parameter files — must be provided at deployment time via `--parameters tenantId=xxx` or CI/CD pipeline variable. Do NOT hardcode tenant IDs.
  </action>
