<task type="auto">
  <name>Task 2: Create shared, networking, and firewall Bicep modules</name>
  <files>infra/modules/shared.bicep, infra/modules/networking.bicep, infra/modules/firewall.bicep</files>
  <read_first>
    - .planning/phases/01-infrastructure-foundation/01-CONTEXT.md (D-10 through D-14 for networking topology, D-19 through D-22 for Bicep organization, D-28 through D-32 for multi-tenant infra foundation)

    - .planning/HLD.md (§10 Network Architecture — hub VNet, spoke VNet, traffic flows)
  </read_first>
  <action>
    Create three Bicep modules:

    **1. infra/modules/shared.bicep** — Naming conventions and shared utilities:
    - Parameters: `environment` (dev/staging/prod), `location`, `projectName` (default: 'aip'), `tags` (object)
    - Output a `naming` object with resource name patterns: `{projectName}-{resourceType}-{environment}` (e.g., `aip-aks-dev`, `aip-kv-dev`)
    - Output a `defaultTags` object merging input tags with: `Environment`, `Project`, `ManagedBy: bicep`, `CreatedAt: utcNow()`
    - Output `location` passthrough for consistency

    **2. infra/modules/networking.bicep** — Hub-spoke VNet topology per D-10:
    - Parameters: `environment`, `location`, `naming` (from shared module output), `tags`
    - **Hub VNet**: address space `10.0.0.0/16`
      - `AzureFirewallSubnet` (`10.0.1.0/24`) — required name for Azure Firewall
      - `AzureBastionSubnet` (`10.0.2.0/24`) — required name for Bastion
      - `GatewaySubnet` (`10.0.3.0/24`) — reserved for VPN/ExpressRoute
    - **Spoke VNet**: address space `10.1.0.0/16`
      - `snet-aks-system` (`10.1.0.0/24`) — AKS system node pool per D-11
      - `snet-aks-user` (`10.1.4.0/22`) — AKS user node pool (/22 for scale) per D-11
      - `snet-aks-gpu` (`10.1.8.0/24`) — Reserved for GPU pool per D-06
      - `snet-apim` (`10.1.16.0/27`) — API Management per D-11
      - `snet-private-endpoints` (`10.1.32.0/24`) — Cosmos DB, Azure SQL, Service Bus private endpoints per D-12
      - `snet-management` (`10.1.48.0/24`) — Management VMs
    - **VNet peering**: Hub ↔ Spoke bidirectional peering with `allowForwardedTraffic: true`
    - **NSGs**: One NSG per subnet with deny-all-inbound default per D-14:
      - `snet-aks-system`: allow internal VNet, deny all inbound
      - `snet-aks-user`: allow from APIM subnet, allow internal VNet, deny all
      - `snet-apim`: allow HTTPS from Front Door service tag, deny all
      - `snet-private-endpoints`: allow from AKS subnets only
    - **Private DNS zones**: Create zones for:
      - `privatelink.documents.azure.com` (Cosmos DB)
      - `privatelink.vaultcore.azure.net` (Key Vault)
      - `privatelink.azurecr.io` (ACR)
      - `privatelink.servicebus.windows.net` (Service Bus)
      - `privatelink.database.windows.net` (SQL)
      - `privatelink.redis.cache.windows.net` (Redis)
    - Link all private DNS zones to spoke VNet
    - Outputs: all subnet resource IDs, VNet IDs, NSG IDs, private DNS zone IDs

    **3. infra/modules/firewall.bicep** — Azure Firewall per D-10:
    - Parameters: `environment`, `location`, `naming`, `tags`, `firewallSubnetId` (from networking module)
    - Public IP for Azure Firewall (Standard SKU, static allocation)
    - Azure Firewall resource (Standard tier):
      - Application rule collection: allow HTTPS to `*.microsoft.com`, `*.azure.com`, `*.github.com`, `*.docker.io`, `*.npmjs.org` (for package management)
      - Network rule collection: allow DNS (53/UDP), HTTPS (443/TCP), NTP (123/UDP) outbound
      - Default action: deny all other egress
    - Route table for spoke subnets (UDR through firewall: `0.0.0.0/0` → firewall private IP)
    - Diagnostic settings output placeholder (connected in monitoring module)
    - Outputs: firewall private IP, public IP, route table ID
  </action>
