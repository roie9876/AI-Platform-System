    Create `infra/modules/aks.bicep` — AKS cluster per CONTEXT.md decisions:

    **Parameters:**
    - `environment` (string)
    - `location` (string)
    - `naming` (object — from shared module)
    - `tags` (object)
    - `aksSystemSubnetId` (string — from networking module)
    - `aksUserSubnetId` (string — from networking module)
    - `logAnalyticsWorkspaceId` (string — optional, for monitoring integration in Plan 03)
    - `acrId` (string — optional, for ACR pull role assignment)

    **AKS Cluster** (`Microsoft.ContainerService/managedClusters`):
    - Kubernetes version: `1.30` (latest stable per D-05 equivalent)
    - DNS prefix: `{naming.prefix}-aks`
    - Identity: SystemAssigned per D-09
    - SKU tier: Standard (uptime SLA) per D-04
    - Network profile per D-07:
      - networkPlugin: `azure`
      - networkPluginMode: `overlay` (CNI Overlay per D-07)
      - networkPolicy: `calico`
      - serviceCidr: `172.16.0.0/16`
      - dnsServiceIP: `172.16.0.10`
      - podCidr: `192.168.0.0/16` (overlay pod CIDR, not from VNet)
    - OIDC issuer: enabled (for workload identity per D-09)
    - Workload identity: enabled per D-09
    - Azure Policy add-on: enabled
    - Microsoft Defender for Containers: enabled
    - Auto-upgrade channel: `patch`

    **System Node Pool** (`agentPoolProfiles` inline):
    - Name: `system`
    - VM size: `Standard_D4s_v5` per D-06
    - Count: 2 (min), max: 4 (autoscale enabled)
    - Mode: System
    - OS: Linux
    - Subnet ID: `aksSystemSubnetId` parameter
    - Taints: `CriticalAddonsOnly=true:NoSchedule`
    - Availability zones: [1, 2, 3]

    **User Node Pool** (separate `Microsoft.ContainerService/managedClusters/agentPools`):
    - Name: `userpool1`
    - VM size: `Standard_D8s_v5` per D-06
    - Count: 2 (min), max: 10 (autoscale enabled)
    - Mode: User
    - OS: Linux
    - Subnet ID: `aksUserSubnetId` parameter
    - Labels: `workload-type: general`
    - Availability zones: [1, 2, 3]

    **Conditional monitoring integration:**
    - If `logAnalyticsWorkspaceId` is provided, enable OMS agent add-on with workspace ID
    - If not provided, skip (will be wired in Plan 03 main.bicep)

    **ACR pull role assignment** (conditional):
    - If `acrId` is provided, create role assignment: AKS kubelet identity → AcrPull role on ACR
    - Use `Microsoft.Authorization/roleAssignments` with `acrPull` role definition ID: `7f951dda-4ed3-4680-a7ca-43fe172d538d`

    **Outputs:**
    - `aksClusterId` (string) — cluster resource ID
    - `aksClusterName` (string) — cluster name
    - `kubeletIdentityObjectId` (string) — for ACR pull and Key Vault access
    - `oidcIssuerUrl` (string) — for workload identity federation
  </action>

    Create two Bicep modules:

    **1. infra/modules/acr.bicep** — Azure Container Registry per STACK.md (ACR Premium):

    Parameters:
    - `environment`, `location`, `naming` (object), `tags` (object)
    - `privateEndpointSubnetId` (string — from networking)
    - `acrPrivateDnsZoneId` (string — from networking `privatelink.azurecr.io`)

    ACR resource (`Microsoft.ContainerRegistry/registries`):
    - SKU: Premium (required for private endpoints and geo-replication)
    - Admin user: disabled (use managed identity instead)
    - Public network access: disabled (private endpoint only per D-12)
    - Zone redundancy: enabled
    - Data endpoint: enabled
    - Policies:
      - Quarantine policy: enabled (for image scanning workflow)
      - Trust policy: enabled (content trust / image signing)
      - Retention policy: enabled, 30 days for untagged manifests

    Private endpoint:
    - `Microsoft.Network/privateEndpoints` linked to ACR
    - Group ID: `registry`
    - Subnet: `privateEndpointSubnetId`
    - Private DNS zone group linking to `acrPrivateDnsZoneId`

    Microsoft Defender for Containers vulnerability scanning:
    - Not a separate resource — rely on subscription-level Defender enablement (document in action notes)

    Outputs:
    - `acrId` (string) — for AKS role assignment
    - `acrLoginServer` (string) — for CI/CD push
    - `acrName` (string) — for reference

    **2. infra/modules/keyvault.bicep** — Azure Key Vault per D-18/ADR-001:

    Parameters:
    - `environment`, `location`, `naming` (object), `tags` (object)
    - `privateEndpointSubnetId` (string — from networking)
    - `kvPrivateDnsZoneId` (string — from networking `privatelink.vaultcore.azure.net`)
    - `aksKubeletIdentityObjectId` (string — optional, from AKS for access policy)
    - `tenantId` (string — Azure AD tenant ID)

    Key Vault resource (`Microsoft.KeyVault/vaults`):
    - SKU: Premium (HSM-backed keys per STACK.md)
    - Tenant ID: `tenantId` parameter
    - Enable RBAC authorization: true (per D-28, use RBAC instead of access policies)
    - Enable soft delete: true, retention days: 90
    - Enable purge protection: true
    - Public network access: disabled (private endpoint only per D-12)
    - Network ACLs: default action deny

    Private endpoint:
    - `Microsoft.Network/privateEndpoints` linked to Key Vault
    - Group ID: `vault`
    - Subnet: `privateEndpointSubnetId`
    - Private DNS zone group linking to `kvPrivateDnsZoneId`

    Role assignment for AKS (conditional on `aksKubeletIdentityObjectId`):
    - Role: Key Vault Secrets User (`4633458b-17de-408a-b874-0445c86b69e6`)
    - Principal: AKS kubelet identity
    - Scope: Key Vault resource

    Outputs:
    - `keyVaultId` (string)
    - `keyVaultUri` (string) — for app configuration
    - `keyVaultName` (string)
  </action>
