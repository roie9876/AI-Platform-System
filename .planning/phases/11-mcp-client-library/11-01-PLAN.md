**Exception classes:**
- `MCPClientError(Exception)` — base error
- `MCPConnectionError(MCPClientError)` — connection/transport failures
- `MCPTimeoutError(MCPClientError)` — timeout exceeded
- `MCPProtocolError(MCPClientError)` — invalid JSON-RPC response or MCP protocol violation

**MCPClient class:**

Constructor `__init__(self, server_url: str, *, timeout: float = 30.0, headers: Optional[Dict[str, str]] = None)`:
- Store server_url, timeout, optional extra headers (for auth tokens)
- Initialize `self._client: Optional[httpx.AsyncClient] = None`
- Initialize `self._session_id: Optional[str] = None` (from Mcp-Session-Id header)
- Initialize `self._request_id: int = 0` (auto-incrementing JSON-RPC id)
- Initialize `self._initialized: bool = False`

Private `_next_id(self) -> int`: Increment and return `_request_id`.

Private `async _ensure_client(self) -> httpx.AsyncClient`: Create httpx.AsyncClient if not exists, with timeout=httpx.Timeout(self._timeout), base headers including `Content-Type: application/json`, `Accept: application/json, text/event-stream`, plus any custom headers. Return client.

Private `async _send_request(self, method: str, params: Optional[dict] = None) -> Any`:
- Build `JsonRpcRequest(id=self._next_id(), method=method, params=params)`
- POST to `self._server_url` with body `request.model_dump(exclude_none=True)`
- Include `Mcp-Session-Id` header if `self._session_id` is set
- On response: check status code (raise MCPConnectionError for non-2xx)
- Check Content-Type header:
  - If `application/json`: parse as JsonRpcResponse or JsonRpcErrorResponse
  - If `text/event-stream`: use httpx_sse to read SSE events, collect the final `message` event containing the JSON-RPC response
- If response is JsonRpcErrorResponse: raise `MCPProtocolError` with error code and message
- Return `response.result`
- Catch `httpx.TimeoutException` → raise `MCPTimeoutError`
- Catch `httpx.HTTPError` → raise `MCPConnectionError`

Private `async _send_notification(self, method: str, params: Optional[dict] = None) -> None`:
- Build `JsonRpcNotification(method=method, params=params)`
- POST to `self._server_url` with body
- Include `Mcp-Session-Id` header if set
- Don't expect a response body (notifications are fire-and-forget)

**Public methods:**

`async connect(self) -> InitializeResult`:
- Call `_send_request("initialize", InitializeParams(...).model_dump())` with clientInfo name="ai-platform", version="1.0.0"
- Capture `Mcp-Session-Id` from response headers if present
- Parse result as `InitializeResult`
- Send `_send_notification("notifications/initialized")` to complete handshake
- Set `self._initialized = True`
- Return InitializeResult

`async list_tools(self, cursor: Optional[str] = None) -> ListToolsResult`:
- Raise MCPClientError if not initialized
- Build params dict, include cursor if provided
- Call `_send_request("tools/list", params)`
- Parse and return as `ListToolsResult`

`async call_tool(self, tool_name: str, arguments: Optional[Dict[str, Any]] = None) -> ToolCallResult`:
- Raise MCPClientError if not initialized
- Call `_send_request("tools/call", ToolCallParams(name=tool_name, arguments=arguments).model_dump())`
- Parse and return as `ToolCallResult`

`async disconnect(self) -> None`:
- If client exists, call `await self._client.aclose()`
- Reset `_client`, `_session_id`, `_initialized`

`async __aenter__` / `async __aexit__`: Context manager support — connect on enter, disconnect on exit.

`@property def is_connected(self) -> bool`: Return `self._initialized`.

Use `logging.getLogger(__name__)` for all logging (consistent with codebase pattern).
